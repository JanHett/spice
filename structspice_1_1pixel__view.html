spice::pixel_view struct

Refers to a single pixel in a spice::image. Note that this class has no information about the size of the image it is referring to and thus cannot perform bounds checking. Use image::at for that. Fulfills concept vector-like.

template<typename T>
spice::pixel_view<T>::pixel_view(T*const data = nullptr, size_t channels = 0) constexpr

Parameters
data Pointer to the first channel of this pixel's data
channels The number of channels in this pixel

Constructs a pixel_view referring to the pixel starting at data. Evidently the view is invalid if a nullptr is passed. Similarly it is the user's responsibility to not refer to the pixel view after the referred-to data has been invalidated. It is therefore recommended to never use this structure explicitly and instead only via image::operator[].

Copies values referenced by other to data array referenced by this pixel_view.

Note

This will copy the first n channels, where n = max(this->size, other.size()).

template<typename T>
T& spice::pixel_view<T>::operator[](size_t channel)

Parameters
channel The index of the channel to retrieve the data from
Returns A single, one-channel data point of the image

Allows accessing pixel data with subscript notation. No bounds checking is performed (use image::at for this purpose).

Note

Accessing pixel values via image::operator() is slightly faster since no intermediate objects have to be constructed. Iterating over image::data is faster still.

Calling my_image[42][47][2] on an RGB image would return the blue channel value in column 43 (index 42), row 48 of the image.
